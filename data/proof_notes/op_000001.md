# Problem op_000001

## Statement
∀ {α : Type u} [SemigroupLike01 α], ∀ x y z : α, (x * y) * z = x * (y * z)

## Attempt Result
counterexample

## Natural Language Sketch
`Fin 3` 上で `op3 x y := if y = 1 then x else if x = y then x else 0` を入れる。これは有限個のケース分けで `SemigroupLike01` の3公理を満たす。だが `x=2, y=1, z=2` で `((2 * 1) * 2) = 2` に対し `2 * (1 * 2) = 0` なので結合律は壊れる。Lean ではこの具体モデルを `letI : SemigroupLike01 (Fin 3)` として与え、公理部分と失敗例を `decide` で閉じればよい。

## Counterexample Intuition
3元集合 `Fin 3 = {0,1,2}` 上で、積を「右引数が `1` なら左引数を返し、そうでなく左右が等しければその値を返し、それ以外は `0` を返す」と定める。この演算は左冪等、公理 `x * (x * y) = x * y`、公理 `(x * y) * z = (x * z) * y` をすべて満たす。しかし `x=2, y=1, z=2` とすると、左辺 `((2 * 1) * 2)` は `2`、右辺 `2 * (1 * 2)` は `0` になる。したがって与えられた結合律は一般には成り立たない。

## Lean Draft
```lean
intro h
let op3 : Fin 3 → Fin 3 → Fin 3 := fun x y => if y = 1 then x else if x = y then x else 0
letI : SemigroupLike01 (Fin 3) :=
  { mul := op3
    ax_left_idempotent := by
      intro x
      change op3 x x = x
      simp [op3]
    ax_right_absorb_duplicate := by
      intro x y
      change op3 x (op3 x y) = op3 x y
      decide
    ax_middle_swap := by
      intro x y z
      change op3 (op3 x y) z = op3 (op3 x z) y
      decide }
have hbad : ¬ (((2 : Fin 3) * 1) * 2 = (2 : Fin 3) * (1 * 2)) := by
  change ¬ (op3 (op3 2 1) 2 = op3 2 (op3 1 2))
  decide
exact hbad (h (α := Fin 3) 2 1 2)
```
